Executive Summary:

Over the past month, the team had been tasked with designing a executable GUI allowing the user to play a game of tic-tac-toe both against the computer or another human player.
Collectively, the group agreed to hone the focus of development toward iterative development using agile development methods. This way, emerging requirements could be
implemented into the game as new versions, with each version providing new functionality or revisions of existing features. Up to this point, the team had developed two
working versions of the project - Alpha 1 and Alpha 2, respectively. All versions have been cataloged for eventual documentation purposes.
The first version, Alpha 1 served as a prototype which included the base functionality for being able to play a game of tic-tac-toe. No respect had been given for allowing the
user to specify the number of games they wanted to play and pre-determined series’ lengths were set to be played in entirety. A simple AI system had been developed in which
the computer would be able to make random moves against the user. All in all, this version was used as a proof of concept and a stimulus for revisions and future requirements
elicitation. Visualizations and more in-depth descriptions for this version are provided below.

Alpha 2 sought to address some of the faults and left out requirements from Alpha 1. Difficulties at this point included dealing with displaying both a game winner and a series
winner to the user without making the view too crowded. This issue was amplified due to the limitation of using a 900 x 900 display and basing all of the current UI decisions around
these dimensions. For example, enlarging the game screen would offset the position of the game board where it is currently anchored. The proposed solution was to add a dedicated
“Game Over” screen which freed the team to work with a fresh, clean panel and avoid confusion from otherwise putting both the game winner and series winner on the same panel.
This solution was similarly reused for the new custom selection screen for game lengths to avoid mixing a buttons screen with a user input display. A full list of fixes and
emergent issues along with visualizations for Alpha 2 are also provided.

The project as a whole has moved out from the proof of concept and basic functional development stages. Progress towards the final version will include UI refinement and movement
towards the development of a smarter AI system. Anticipated difficulties include clarifying what will make a “smart” AI. The group plans to address this issue by developing a hierarchy
of move decisions for the AI so it understands what order it needs to consider its moves before taking an action.

Alpha 1:

This version of the tic-tac-toe project features a “Player vs Player” and a “Player vs Computer” button on the main menu screen with either allowing the user to issue a match
against another human user or the computer, respectively. Selecting either choice will then allow you to select a Best of 3 or Best of 5 match. As of this implementation, the computer
will always make the second move and the entire series will be played out regardless if a winner can already be decided (i.e. in the case where Player 1 has 2 wins in a Best of 3 match,
the third match will still be played). When the winner of a match is declared - or in the event of a tie - the result is displayed directly above the game board. Once the series has concluded,
the button directly below the game board responsible for moving the player between games will change text and functionality, displaying the overall match winner and returning the user(s)
back to the Main Menu screen. Figures 1-3 illustrate the UI for each screen of the game.

Alpha 2:

This version of the tic-tac-toe project features many improvements from the previously released version, Alpha 1. The user is now able to choose a custom
range of games to play via an input select screen after choosing whether it is a 2-player or player versus AI game. Regarding the AI, the computer is now able
to be selected to make the first move, though its piece placement remains random in this version. A series will now terminate early if either player takes
majority wins, and a game over screen has been implemented which displays the game winner in large font at the top of the screen. Two buttons have also been
added to the game over screen: a Play Again button which allows the user to repeat the previously set up match without having to re-select game options, and
a Main Menu button which simply returns the user back to the title screen. Figures 4-7 illustrate the changes made to existing game screens, as well as the
implementation of the new custom game select and game over screens.
